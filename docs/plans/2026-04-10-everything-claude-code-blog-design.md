# Everything Claude Code 博客设计说明

## 目标

新增一篇 AI 工具体验文，主题为 `Everything Claude Code`，但不写成安装教程，而是写成“开源项目体验 + 方法论判断”。

## 选题判断

仓库最有价值的点，不是命令数量，而是它把 `rules`、`skills`、`commands`、`agents`、`hooks`、验证循环和跨工具适配组织成了一套相对完整的 AI Coding 工作流系统。

## 采用角度

采用“它不只是提示词仓库，更像一套工作流操作系统”的切入方式，突出以下三点：

1. 它优化的是整条工作链，而不是单次对话
2. 它强调研究优先、验证优先、持续学习
3. 它试图沉淀一层跨 Claude Code、Codex、Cursor、OpenCode、Gemini 的可迁移工作流

## 边界

- 不伪装成深度实测报告
- 不写成大段安装步骤
- 保留对配置复杂度和适用人群边界的提醒

## 目标读者

对 AI Coding、Agent 工作流、开源工具链有兴趣，且已经开始从“偶尔用一下 AI”走向“把 AI 纳入长期开发流程”的开发者。
